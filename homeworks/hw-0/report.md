# Как работает агент с кодом.

## Проблемы и особенности которые замечал:

1. Без декомпозии и детализированного плана принимает странные решения, переусложняет, иногда пишет очень громоздкие сложные, плохо читаемые конструкции, в ruby использует метапрограммирование без причины. Частично решается прописыванием инструкций в системном промте, но основное - обязательная и очень тщательная стадия планирования задач, особенно больших.

2. При выполнении нескольких достаточно больших задач подряд даже с детализацией плана все равно накапливается "мусор" - неиспользуемые методы, которые забыл удалить, явно лишние, создающие только дополнитеную обертку для вызовов и дублирующие. Если идет добавление фич, архитектура остается прежней перегружает/переусложняет классы/методы.
Решается периодическими задачами на ревью/рефакторинг (можно добавить скиллы на них) - замечает/исправляет почти все. Также неплохо делать кросс-ревью codex-claude, ревью/рефакторинг после выполнение подзадачи и отдельно по мере выполнения шага уже в связке с предыдущим.

3. Строгая типизация дает улучшение потому лучше использовать ts, rbs.
