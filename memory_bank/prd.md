# Product Requirements Document — LLM Learning System

## Продукт

Система структурированного обучения с помощью AI. Пользователь описывает тему через диалог, система генерирует воспроизводимый конспект и набор учебных материалов на его основе.

## Для кого

Один пользователь — разработчик проекта. Multi-user и аутентификация вне scope.

## Ключевые продуктовые области

### 1. Intake и генерация конспекта
Пользователь ведёт диалог, уточняя параметры темы. Система проверяет полноту и переводит диалог в структурированный brief. На основе brief собирается финальный prompt и генерируется конспект. Конспект сохраняется на диске.

Workflow: `collecting_info → intake_complete → brief_ready → prompt_ready → conspect_generated`

### 2. Итерация над конспектом
После первой генерации система проводит self-review. Если конспект требует доработки — пользователь может запустить регенерацию в рамках того же диалога без потери контекста.

### 3. Учебные материалы на основе конспекта (будущее)
На основе конспекта система генерирует:
- Карточки и flashcards
- Тесты и вопросы на понимание
- Краткие выжимки

### 4. Режимы обучения (будущее)
- Диалоговое обучение в рамках конспекта
- Метод Фейнмана
- Режим экзаменатора

### 5. Границы конспекта (будущее)
Конспект работает как ограничение объёма: система либо остаётся строго в его рамках, либо при необходимости дополняет тему из интернета.

## Не в scope (текущая итерация)

- Аутентификация и разграничение данных по пользователям
- UI файлового навигатора и редактора конспектов
- Пагинация и поиск по диалогам
- Стриминг ответа LLM по частям
- Интеграция с конкретными локальными моделями

## Текущее состояние реализации

Реализованы области: сохранение диалогов (001), файловый контракт (002), workflow state диалога (003).
В работе: prompt templates (004), runner (005), intake check (006), brief generation (007), prompt builder (008), conspect generation (009), self-review loop (010).

## Технические принципы

- **File-first** для учебных артефактов: конспекты — документы, не CRUD
- **ActionCable** для real-time диалога с LLM
- **RubyLLM** как unified LLM-клиент
- **Rails MVC** без преждевременных абстракций
