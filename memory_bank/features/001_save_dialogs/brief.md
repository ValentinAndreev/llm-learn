## Цель

Пользователь теряет историю диалога при перезагрузке страницы и не может вернуться к предыдущим разговорам. Нужно обеспечить, чтобы пользователь мог закрыть браузер и выбрать любой из предыдущих диалогов из списка — и продолжить разговор без потери истории.

## Для кого

Пользователь приложения, который ведёт повторяющиеся или многосессионные диалоги с LLM в рамках обучения.

## Доменный контекст

Базовое диалоговое окно реализовано (фича hw-0). LLM не помнит контекст прошлых сессий — каждый раз начинается с нуля.

## Технический контекст

Rails + ActionCable (`ChatChannel` обрабатывает отправку сообщений и получение ответов LLM), Stimulus (`chat_controller.js` управляет UI), `ChatController` отображает страницу чата. БД пока не используется для хранения диалогов.

## Требования

- Текущий чат не сохраняется между сессиями — при перезагрузке страницы история теряется полностью. Пользователь должен иметь возможность выбрать любой предыдущий диалог из списка и продолжить его с того места, где остановился.
- Пользователь может управлять своими диалогами: переименовывать и удалять их.

## Что НЕ в скоупе

- Аутентификация и разграничение диалогов по пользователям
- Поиск и фильтрация по диалогам
- Пагинация списка диалогов
- Стриминг ответа LLM по частям
