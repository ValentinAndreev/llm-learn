## Цель

Сейчас невозможно ассоциировать диалог с одной темой и отслеживать его стадию: при возврате в диалог система не знает цель процесса, текущий шаг и актуальный prompt. Нужно хранить состояние workflow и привязку к теме внутри диалога.

## Для кого

Для пользователя, который ведёт один диалог для одной темы и хочет накапливать цель, состояние процесса и текущую версию итогового prompt.

## Доменный контекст

Сейчас `Dialog` хранит только заголовок и сообщения. Для нового workflow в диалоге должны жить цель, текущий статус процесса и актуальный prompt для генерации конспекта.

## Технический контекст

`Dialog` и `Message` уже существуют и используются через `ChatChannel`, `ChatController`, `DialogsController` и Stimulus UI.

## Требования

- Диалог должен хранить цель работы над темой и сохранять её между сессиями.
- Диалог должен хранить текущую стадию workflow и иметь явно заданное начальное состояние для нового диалога.
- Диалог должен хранить идентификатор темы после первой успешной генерации конспекта.
- Диалог должен хранить актуальную версию итогового prompt для генерации конспекта.
- Эти данные должны быть доступны через существующий Rails MVC без добавления отдельного слоя абстракции.

## Что НЕ в скоупе

- История версий prompt
- Оценки качества конспекта
- Аналитика по обучению
