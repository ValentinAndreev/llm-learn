## Цель

Сейчас prompt-тексты для LLM-сценариев неизбежно будут размазаны по Ruby-коду, из-за чего их трудно редактировать, сравнивать и ревьюить отдельно от логики. Нужен файловый слой шаблонов, который можно поддерживать как обычные документы.

## Для кого

Для разработчика проекта и для будущего workflow, в котором prompt templates редактируются как файлы, а не хардкодятся строками в Ruby.

## Доменный контекст

Пользователь сначала ведёт диалог для сбора информации о теме, затем система проверяет полноту данных, формирует итоговый prompt на конспект и позже проверяет результат.

## Технический контекст

Сейчас в проекте нет каталога prompt templates и нет file-first слоя для них. Вызов LLM идёт напрямую через RubyLLM.

## Требования

- Должны существовать отдельные markdown templates для каждого LLM-сценария, участвующего в intake-процессе и генерации конспекта.
- Для каждого template должно быть понятно его назначение и ожидаемый формат ответа модели.
- Все templates должны храниться в одном месте, согласованном с файловым контрактом проекта.
- Приложение не должно зависеть от хардкода этих prompt-текстов в Ruby-коде.

## Что НЕ в скоупе

- Выполнение templates через LLM
- UI для редактирования templates
- Версионирование templates
