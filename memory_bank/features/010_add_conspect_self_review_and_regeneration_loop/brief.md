## Цель

После первой генерации пользователь может остаться недоволен результатом, но без итерационного цикла ему придётся начинать новый диалог и заново накапливать контекст. Нужно проверять каждый сгенерированный конспект и позволять повторную генерацию в рамках того же диалога.

## Для кого

Для пользователя, которому нужен не просто первый черновик, а возможность улучшать конспект без потери накопленного контекста.

## Доменный контекст

После генерации конспекта система должна сама оценить результат, показать возможные пробелы и дать возможность продолжить тот же диалог, уточнить prompt и получить новую версию.

## Технический контекст

Диалог уже сохраняет контекст, итоговый prompt уже хранится в `Dialog`, а тема уже создана в `learning/topics/`.

## Требования

- Self-review должен запускаться автоматически сразу после каждой генерации конспекта.
- Результат проверки должен сохраняться как отдельный артефакт темы.
- Пользователь должен иметь возможность продолжить тот же диалог, пересобрать prompt и запустить повторную генерацию.
- При повторной генерации предыдущая версия конспекта не должна теряться.
