## Цель

Накопленная в диалоге информация лежит в raw messages, и из неё нельзя извлечь параметры темы без ручного разбора сообщений. Нужна промежуточная структура, которая переводит диалог в структурированный набор параметров, который можно передать в prompt template без дополнительной обработки.

## Для кого

Для пользователя и для системы, чтобы накопленная в диалоге информация не жила только в raw messages, а переводилась в управляемую структуру.

## Доменный контекст

После достаточного сбора информации система должна выделить из диалога цель, границы темы, уровень сложности, ограничения, смежные темы и ожидаемый результат.

## Технический контекст

Диалог и сообщения уже сохраняются в БД. После проверки полноты можно запускать отдельный LLM-сценарий на сборку brief.

## Требования

- На основе цели и истории сообщений должен формироваться brief с полями: цель, границы темы, уровень сложности, ограничения, смежные темы, ожидаемый результат.
- Brief должен быть представлен в структурированном формате, например YAML или JSON, который можно передать как переменные в prompt template.
- Brief должен быть связан с текущим диалогом и доступен следующему шагу workflow.
- После успешной сборки brief стадия workflow диалога должна обновляться.

## Что НЕ в скоупе

- Сохранение brief в `learning/topics/`
- Генерация конспекта
- Пользовательское редактирование brief в UI
