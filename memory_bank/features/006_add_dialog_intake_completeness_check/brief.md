## Цель

Сейчас система не умеет определить, готов ли диалог к генерации сфокусированного конспекта, поэтому пользователь либо получает сырой результат слишком рано, либо вручную гадает, чего ещё не хватает. Нужно ввести шаг проверки полноты входных данных.

## Для кого

Для пользователя, который не хочет сразу получать сырой и плохо сфокусированный конспект без уточнения ширины, глубины и уровня сложности темы.

## Доменный контекст

Диалог используется для сбора информации о теме. Прежде чем строить prompt на конспект, нужно понять, хватает ли вводных данных и чего именно не хватает.

## Технический контекст

В проекте уже есть сохранённые диалоги и сообщения. После добавления prompt runner можно запускать отдельный template на анализ содержимого диалога.

## Требования

- Система должна анализировать цель диалога и накопленную историю сообщений перед переходом к сборке финального prompt.
- Система должна определять, хватает ли информации для генерации сфокусированного конспекта.
- Если информации не хватает, система должна формировать перечень недостающих данных, понятный для следующего шага workflow.
- Результат проверки должен быть доступен для следующего шага workflow; конкретный способ хранения определяется реализацией.
- Стадия workflow диалога должна обновляться по результату проверки.

## Что НЕ в скоупе

- Сборка итогового prompt
- Генерация конспекта
- Сохранение файлов темы
