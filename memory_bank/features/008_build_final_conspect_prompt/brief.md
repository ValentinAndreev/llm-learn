## Цель

Без одного сохранённого prompt генерация конспекта для одной и той же темы не воспроизводима: LLM каждый раз получает разный контекст из диалога. Нужен один воспроизводимый prompt, который можно сохранить и пересобирать после уточнений.

## Для кого

Для пользователя, который хочет получить воспроизводимый prompt, а не случайный ответ LLM по последним сообщениям.

## Доменный контекст

После intake и brief система должна иметь актуальный prompt, по которому можно многократно генерировать и уточнять конспект для одной темы.

## Технический контекст

В `Dialog` уже есть поле `current_conspect_prompt`, а prompt runner и базовые templates должны позволять построить итоговый prompt отдельным шагом.

## Требования

- Система должна формировать один итоговый prompt для генерации конспекта из brief и актуального контекста диалога.
- Итоговый prompt должен сохраняться в диалоге между сессиями.
- После продолжения диалога итоговый prompt должен пересобираться с учётом нового контекста.
- После успешной сборки prompt стадия workflow диалога должна переходить в состояние готовности к генерации конспекта.

## Что НЕ в скоупе

- Выполнение итогового prompt
- Сохранение темы в файловой системе
- Версионирование prompt
