## Цель

После завершения intake-процесса у пользователя нет материального результата: тема существует только в диалоге и в БД. Нужно, чтобы успешно завершённый workflow создавал осязаемый артефакт на диске.

## Для кого

Для пользователя, который завершил intake-диалог и хочет получить первый материальный результат в виде темы на диске.

## Доменный контекст

Один диалог соответствует одной теме. Папка темы должна появляться только тогда, когда уже есть реальный результат генерации, а не заранее.

## Технический контекст

`Dialog` хранит `topic_slug` и `current_conspect_prompt`, а структура `learning/topics/` должна быть описана заранее отдельным контрактом.

## Требования

- После успешной генерации первого конспекта должен создаваться файловый каталог темы согласно контракту хранения из задачи 002.
- Файловый каталог темы не должен создаваться до успешной генерации.
- В файловый каталог темы должны сохраняться артефакты первой версии, предусмотренные контрактом хранения из задачи 002.
- Созданная тема должна быть связана с исходным диалогом.

## Что НЕ в скоупе

- Self-review конспекта
- Регенерация и версии
- Definitions и flashcards
