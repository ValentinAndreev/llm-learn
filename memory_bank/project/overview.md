# Project Overview

## Что строим

**LLM Learning System** — приложение для структурированного обучения с помощью AI.

Пользователь ведёт intake-диалог с LLM, уточняя параметры темы. По завершении система генерирует структурированный конспект, который сохраняется на диске. На основе конспекта и связанных материалов система генерирует карточки, тесты, краткие выжимки и вопросы на понимание. Поддерживается диалоговое обучение, режим проверки знаний (метод Фейнмана, экзаменатор).

Архитектурно — набор независимых LLM-сценариев вокруг общего ядра знаний (конспекта). Отдельный слой интеграции для работы с разными API и локальными моделями.

**Стек:** Rails 8.1, Ruby 4.0.1, PostgreSQL, ActionCable, Stimulus, RubyLLM, RBS+Steep.

## Ключевые принятые решения

| Решение | Почему |
|---|---|
| Rails MVC, без service objects | Достаточно для текущей сложности; усложнение только по явному запросу |
| ActionCable для LLM-диалога | Real-time стриминг ответов; архитектура уже сложилась в hw-0 |
| File-first для учебных артефактов | Конспекты — это документы, не CRUD-сущности; `learning/` на диске |
| БД только для Dialog и Message | Сессионные данные и workflow state — в Postgres; результаты — в файлах |
| RBS + Steep для типизации | Дисциплина по сигнатурам; добавлять RBS для каждого нового класса и метода |
| RubyLLM как LLM-клиент | Unified интерфейс для разных провайдеров |

## Файловая структура (runtime-артефакты)

```
learning/
  prompts/                    ← markdown-шаблоны промптов для LLM-сценариев
  topics/<topic_slug>/        ← создаётся ТОЛЬКО после первой успешной генерации
    meta.yml                  ← topic_slug, source_dialog_id, created_at, current_state
    brief.md                  ← структурированные параметры темы из intake
    conspect_prompt.md        ← финальный промпт для генерации конспекта
    conspects/
      current.md              ← актуальный конспект
      versions/               ← история версий
    reviews/
      history/                ← история ревью
```

## Что НЕ строим

- Аутентификация и multi-user (диалоги без разграничения по пользователям)
- Поиск и фильтрация по диалогам
- Пагинация списка диалогов
- Стриминг ответа LLM по частям
- UI файлового навигатора и редактора
- Генерация карточек, flashcards, definitions (пока не в scope реализованных фич)
- CRUD-интерфейс для управления конспектами через UI
